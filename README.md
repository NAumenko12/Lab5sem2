# Задание 1. Учет товаров на складе

## Условие

Нужно реализовать программу для учета товаров на складе.

Склад состоит из ячеек с адресами. Адрес состоит из:

- зоны хранения: `А`, `Б`, `В`;
- номера стеллажа;
- номера вертикальной секции;
- номера полки.

Вариант 8:

- количество зон: `3`;
- количество стеллажей в каждой зоне: `14`;
- количество вертикальных секций стеллажа: `4`;
- количество полок в вертикальной секции: `8`;
- максимальная вместимость одной ячейки: `10` единиц товара;
- общая вместимость склада: `13440` единиц товара.

Поддерживаются команды:

```text
ADD товар количество адрес
REMOVE товар количество адрес
INFO
```

Примеры адресов для варианта 8:

```text
А0111
А1428
Б0345
В1012
```

## Тест 1. Добавление товара

### Ввод

```text
ADD Apples 5 А0111
```

### Ожидаемый результат

```text
Добавлено 5 Apples в А0111
```

## Тест 2. Ячейка занята другим товаром

### Ввод

```text
ADD Apples 5 А0111
ADD Pears 5 А0111
```

### Ожидаемый результат

```text
Добавлено 5 Apples в А0111
Ячейка занята
```

## Тест 3. Превышение вместимости ячейки

### Ввод

```text
ADD Apples 5 А0111
ADD Apples 6 А0111
```

### Ожидаемый результат

```text
Добавлено 5 Apples в А0111
Превышена вместимость
```

## Тест 4. Удаление товара

### Ввод

```text
ADD Apples 5 А0111
REMOVE Apples 3 А0111
```

### Ожидаемый результат

```text
Добавлено 5 Apples в А0111
Удалено 3 Apples (остаток: 2)
```

## Тест 5. Ошибки при удалении

### Ввод

```text
ADD Apples 5 А0111
REMOVE Apples 6 А0111
REMOVE Pears 1 А0111
```

### Ожидаемый результат

```text
Добавлено 5 Apples в А0111
Недостаточно товара
Товар не найден
```

## Тест 6. Информация о складе

### Ввод

```text
ADD Apples 5 А0111
INFO
```

### Ожидаемый результат начинается так

```text

Загруженность зоны А: 0.11 %

```

После строки `Пустые ячейки:` программа выводит длинный список всех свободных адресов.

# Задание 2. Управление контейнерами в порту

## Условие

Нужно реализовать программу для моделирования работы порта по приему и отгрузке контейнеров.

Контейнеры прибывают в порт последовательно и размещаются в стеки. Каждый стек имеет ограничение по максимальному весу. Контейнеры размещаются по принципу `LIFO`: последний пришел, первый ушел.

Поддерживаются команды:

```text
ARRIVE идентификатор вес
LOAD
```

Команда `ARRIVE` добавляет контейнер в первый подходящий стек. Если подходящего стека нет, создается новый стек.

Команда `LOAD` распределяет контейнеры по секциям судна. Контейнеры загружаются в обратном порядке прибытия. Каждый следующий контейнер помещается в секцию с наименьшим текущим весом.

Перед командами программа запрашивает:

- максимальный размер стека;
- количество секций судна.

## Тест 1. Пример из задания

### Ввод

```text
20
2
ARRIVE C001 5
ARRIVE C002 8
ARRIVE C003 7
ARRIVE C004 10
LOAD
```

### Ожидаемый результат

```text
Введите максимальный размер стека: 
 Введите количество секций судна: 
 Контейнер C001 размещен в стек 1
Контейнер C002 размещен в стек 1
Контейнер C003 размещен в стек 1
Контейнер C004 размещен в стек 2
Секция 1 15 тонн: C004 C001 
Секция 2 15 тонн: C003 C002 
```

## Тест 2. Создание нового стека

### Ввод

```text
10
2
ARRIVE C001 6
ARRIVE C002 6
LOAD
```

### Ожидаемый результат

```text
Введите максимальный размер стека: 
 Введите количество секций судна: 
 Контейнер C001 размещен в стек 1
Контейнер C002 размещен в стек 2
Секция 1 6 тонн: C002 
Секция 2 6 тонн: C001 
```

## Тест 3. Распределение по трем секциям

### Ввод

```text
15
3
ARRIVE C001 5
ARRIVE C002 5
ARRIVE C003 5
ARRIVE C004 5
LOAD
```

### Ожидаемый результат

```text
Введите максимальный размер стека: 
 Введите количество секций судна: 
 Контейнер C001 размещен в стек 1
Контейнер C002 размещен в стек 1
Контейнер C003 размещен в стек 1
Контейнер C004 размещен в стек 2
Секция 1 10 тонн: C004 C001 
Секция 2 5 тонн: C003 
Секция 3 5 тонн: C002 
```

## Тест 4. Неизвестная команда

### Ввод

```text
20
2

```

### Ожидаемый результат


# Задание 3. Самолеты и города

## Условие

Программа хранит информацию о самолетах и городах, в которые они летают.

Поддерживаются команды:

```text

PLANES_FOR_TOWN город
TOWNS_FOR_PLANE самолет
PLANES
```

## Тест 1. Создание самолетов и общий список

### Ввод


# Задание 4. Расписание занятий по дням месяца

## Условие

Программа хранит расписание дисциплин по дням месяца.

Поддерживаются команды:

```text
CLASS день дисциплина
VIEW день
NEXT
```

Команда `CLASS` добавляет дисциплину на указанный день.

Команда `VIEW` показывает занятия в указанный день.

Команда `NEXT` выполняет переход к следующему месяцу. Если в новом месяце меньше дней, занятия с лишних дней переносятся на последний день нового месяца.

Перед командами программа считывает количество запросов `N`.

## Тест 1. Повторяющаяся дисциплина

### Ввод

```text
4
CLASS 5 INFORMATICA
CLASS 5 INFORMATICA
VIEW 5
VIEW 1
```

### Ожидаемый результат

```text
Введите количество запросов (N):
Добавлена дисциплина INFORMATICA на день 5
Ошибка: Дисциплина INFORMATICA уже есть в этот день
В день 5 занятия в университете: INFORMATICA
В день 1 мы свободны!
```

## Тест 2. Перенос с 30-го дня в февраль

### Ввод


### Ожидаемый результат

```text

Переход на следующий месяц
В день 28 занятия в университете: YAP
Ошибка: В этом месяце всего 28 дней
```

## Тест 3. Несколько дисциплин в один день
